Long-term Memory

长期记忆系统

AIRI Gemma 的核心不是普通聊天记录，而是一个面向长期陪伴和私人实验的记忆系统：把用户画像、偏好、项目状态、边界规则和可检索知识库沉淀为可审查、可迁移、可持续演化的本地上下文。

双脑系统 LLMWiki
记忆不只是向量召回，而是一条从候选记忆、隐私分级、审查确认到长期知识库的完整链路。
Dual Brain

双脑载入：快反应与长期人格分开管理

你主要开发的长期记忆系统可以理解为 AIRI 的第二个大脑：即时对话仍然保持轻量、快速、可中断；长期脑负责沉淀稳定事实、人物设定、项目状态和用户边界。这样角色不会每次重新认识用户，也不会把所有历史粗暴塞进上下文窗口。

双脑系统的关键是“载入”而不是“堆叠”：对话时只把本轮真正相关、通过安全规则的记忆片段装入上下文；完整记忆库仍然留在本地，等待检索、审查和演化。

Fast Brain

即时对话脑

处理当前输入、语音、视觉和角色反馈，保证桌面舞台里的 AIRI 能及时回应。

Memory Brain

长期记忆脑

保存长期偏好、人物设定、项目线索、边界规则和可复用知识，按隐私等级决定能否进入本轮上下文。

Safety Gate

隐私与安全闸门

Secret 记忆、敏感路径、密钥和不适合云端模型的内容会被拦截，只在本地链路中可控使用。

LLMWiki

把碎片记忆整理成可读知识库

LLMWiki 不是把聊天原文扔进向量库，而是把稳定、可公开或可本地审查的记忆编译成 Markdown Wiki。人可以读，模型可以检索，也能迁移到 Obsidian 风格的知识库。

Raw Sources聊天、文档、网页剪藏和项目资料先作为事实来源进入导入管线，不直接覆盖长期记忆。
Review候选记忆经过安全扫描、隐私分级和人工确认，避免把敏感内容写入公共上下文。
Memory DB本地数据库仍是事实源，保存状态、重要性、类型、更新时间和检索标记。
LLMWiki导出用户画像、项目状态、边界规则、index.md 和 log.md，形成可读、可审计的长期脑。
Memory Lifecycle

从一次对话到长期记忆

这条链路让 AIRI 的记忆有来源、有权重、有衰减、有审查，也能在需要时重新进入上下文。

1

捕获

从对话、任务、视觉或项目资料中抽取可能有长期价值的事实和偏好。

2

筛选

识别 secret、public、local 等隐私等级，过滤密钥、账号和危险指令。

3

确认

进入 Review Workbench，只有被确认的候选记忆才会成为 active 记忆。

4

检索

根据当前任务召回相关片段，按目标模型决定哪些内容可以载入。

5

演化

通过衰减、冲突检查、梦境整理和 LLMWiki 导出，让记忆长期保持干净。

Innovation

主要创新点

这部分适合作为你对外介绍 AIRI Gemma 时的核心叙事：角色、桌面舞台和本地模型只是入口，长期记忆系统才是让 AIRI 变成“个人智能体”的底层能力。

01

本地优先的长期脑

长期偏好、用户边界和项目状态优先保存在本地，云端模型默认只能接触通过 public-safe 规则的上下文。

02

双脑按需载入

即时对话脑负责实时互动，长期记忆脑负责稳定人格和知识。每轮只载入相关片段，避免上下文污染。

03

LLMWiki 知识库

把 Memory DB 中稳定、可审查的内容导出为 Markdown Wiki，并生成 index.md 与 log.md，方便检索、迁移和审计。

04

记忆衰减与情绪权重

通过重要性、检索次数、更新时间、情绪倾向等信号，让 AIRI 的长期状态更接近真实陪伴。

05

安全扫描与隐私闸门

Prompt injection、密钥、敏感路径和 secret 记忆会被 withheld，保证记忆不会因为一次检索被错误暴露。

06

梦境整理工作流

在长期运行后，由本地模型整理可回写的候选记忆、过期结论和冲突项，形成可确认的记忆演化报告。
