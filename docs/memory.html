GBrain Memory Stack

自有记忆知识库

AIRI Gemma 的长期记忆系统以 GBrain 为核心，把 LLMWiki、Obsidian、Hermes 记忆治理、Claude / Codex 协作整理和本地 Memory DB 串成一套可审查、可迁移、可进化的个人知识库。

技术栈 双脑系统
目标不是简单保存聊天记录，而是把长期事实、偏好、项目状态和安全边界沉淀成 AIRI 能理解、人也能审查的 GBrain。
Technical Stack

LLMWiki + GBrain + Obsidian + Hermes

这一页展示的是你主导开发的长期记忆技术栈：从原始资料进入本地记忆，再被整理为可读 Wiki、可搜索上下文和可持续进化的个人知识库。

G

GBrain 核心脑

作为 AIRI 的自有记忆知识库，承载用户画像、项目状态、长期偏好、边界规则和角色经验，而不是把记忆散落在聊天窗口里。

W

LLMWiki 编译层

把稳定记忆导出成 Markdown Wiki，生成 index.md、log.md、项目页和画像页，让人类、模型和后续训练链路都能复用。

O

Obsidian 知识库

支持本地 vault 导入与导出，用文件树、反链、标签和 Markdown 阅读体验承载长期知识，而不锁死在单一应用里。

H

Hermes 记忆治理

借鉴 Hermes 的 MEMORY / USER 摘要、外部 Memory Provider、容量治理、纠错和工具化记忆操作。

C

Claude / Codex 协作

用于整理文档、审查候选记忆、生成知识库 diff 和维护工程任务，但私有事实仍由本地记忆库与用户确认控制。

M

Memory DB 事实源

PGlite / 本地 Memory Service 保存 active、needs_review、privacy、importance、accessCount 等结构化状态。

Dual Brain System

双脑系统：即时反应脑 + 长期记忆脑

双脑系统是 AIRI 记忆架构里最关键的表达：一个脑负责当下互动，一个脑负责长期知识和人格连续性，中间由安全载入层决定什么能进入本轮上下文。

01

即时反应脑

处理当前对话、语音、视觉、桌面舞台状态和角色反馈，保证 AIRI 在使用时能快速回应、保持情绪和动作连续。

02

长期记忆脑

由 GBrain、Memory DB、LLMWiki 和 Obsidian vault 组成，保存用户画像、项目状态、长期偏好、边界规则和可复用知识。

03

安全载入闸门

每轮只把命中的有限摘要和安全片段载入上下文；secret、sensitive、prompt injection 和不适合云端的内容会被 withheld。

快反应适合当前任务、当前语气、舞台反馈和短期上下文，不承担完整历史记忆。
长记忆负责跨会话连续性，让 AIRI 不必每次重新认识用户和项目。
按需载入不是把全部历史塞进 prompt，而是根据本轮任务检索、过滤、压缩后载入。
可解释能追踪使用了哪些记忆、为什么使用、哪些内容被拦截，方便审查和纠错。
Architecture

完整自有记忆知识库架构

GBrain 的核心思路是把“可被模型使用的上下文”和“可被人审查的知识库”放在同一条链路里。Memory DB 是事实源，LLMWiki 是可读视图，Obsidian 是本地知识库界面，Hermes 模式提供记忆治理规则。

每轮对话只载入有限摘要和命中的片段；完整知识库留在本地，通过隐私等级和安全扫描决定能否进入 AIRI、Claude、Codex 或云端模型上下文。

Source Layer

资料源

聊天记录、Markdown、项目文档、网页剪藏、Obsidian vault 和手动输入。

Memory Layer

本地记忆库

候选记忆、长期事实、隐私等级、重要性、访问次数和最近使用时间。

Wiki Layer

LLMWiki / GBrain

把稳定内容组织成用户画像、项目状态、边界规则、时间线、index.md 和 log.md。

Agent Layer

AIRI / Claude / Codex

按需读取摘要和安全片段，用于陪伴、工程协作、知识整理和自我纠错。

Hermes Evolution

Hermes 记忆进化系统

这里不是照搬 Hermes，而是把它的长期记忆治理思想落到 AIRI：有限摘要、外部 Provider、纠错、审查、压缩和自我改进。

1

Ingest

从聊天、文档和 vault 中抽取候选记忆，默认进入 needs_review。

2

Safety

扫描 prompt injection、密钥、隐私路径和敏感内容，决定 public / local / sensitive / secret。

3

Review

通过 MemoryReviewWorkbench 做确认、拒绝、合并、归档和重新分级。

4

Compact

生成 USER_PROFILE、AIRI_MEMORY 和 PROJECT_CONTEXT 摘要，控制每轮上下文体积。

5

Evolve

通过纠错、衰减、冲突检测、梦境整理和 LLMWiki 导出持续更新 GBrain。

Memory Tools

从“会存”到“会被纠正”

长期记忆真正难的地方不在保存，而在修改、解释、撤销、合并和避免错误记忆长期污染角色。

memory.add_candidate新增候选记忆，但不直接污染长期脑。
memory.approve用户确认后进入 active，可被 AIRI 检索和摘要。
memory.replace用于纠错和合并，把过期事实替换为更新版本。
memory.explain_usage解释为什么本轮使用了某条记忆，让系统可观察、可追责。

记忆检索

从 GBrain 和 Memory DB 中取回与当前任务相关的片段，而不是暴力拼接全部历史。

记忆衰减

旧记忆会随时间、访问次数和重要性变化逐步降权，减少过期上下文污染。

情绪权重

为陪伴型 Agent 留下更细腻的状态表达，让记忆不只是事实条目。

记忆控制器

将候选、检索、导出、清理和审查流程收束成可管理的控制界面。
